# AgentCanvas — 让 AI Agent 的思考过程看得见

> 给 Agent 一个任务，看它在一棵交互式树形图里一步一步思考。

## 痛点在哪

AI Agent 能力很强，但像个黑盒。它怎么拆任务的、调了哪个工具、为什么选 A 不选 B、出了错怎么扳回来的——这些你全看不见。不管你是想搞懂 Agent 怎么运作，还是自己正在开发 Agent，都只能靠猜。

**AgentCanvas** 把这层黑布揭开了。Agent 跑任务的整个过程，变成一棵能点开细看的树形图。每一步在想什么、做了什么、结果怎样，清清楚

楚。 

## 能展示什么

| 环节 | 看得见什么 |
|------|-----------|
| **任务拆解** | Agent 怎么把一个大目标切成一个个小步骤 |
| **推理链条** | 每步它脑子里在想什么、为什么这么想 |
| **工具调用** | 调了哪个工具、传了什么参数、返回了什么 |
| **决策分叉** | 为什么走 A 路线不走 B，走错了怎么退回来 |
| **纠错过程** | 发现执行出错了，怎么调整策略重新来 |
| **教学模式** | 一步一步播放，每步之间暂停，适合边看边讲 |
| **分层视图** | 新手看简版，开发者看技术细节 |
| **多视图切换** | 树形图、时间线、流程图，随便换 |

## 长什么样

Agent 执行过程大概是这样一棵树：

```
任务：「调研最好的 Go Web 框架」
├── 🔍 搜索：「Go web 框架 2026」→ 搜到 15 条
│   ├── 📖 打开 gin-gonic.com → 看了特性
│   ├── 📖 打开 echo.labstack.com → 看了特性
│   └── 📖 打开 fiber.wiki → 看了特性
├── 🤔 分析：比比哪个好……
│   ├── 🏷️ Gin：Star 最多，用的人多
│   ├── 🏷️ Echo：轻量，跑得快
│   └── 🏷️ Fiber：写法像 Express，上手快
└── ✅ 结论：生产环境用 Gin，写微服务用 Echo
```

## 技术选型

| 环境 | 后端 | 数据库 | 前端 |
|------|------|--------|------|
| 线上（Oxelia51） | Go | PostgreSQL | React + D3.js |
| 桌面（exe） | Go | SQLite | 内嵌 React + D3.js |

## 数据哪来

- **先做**：内置几个演示 Agent 和预设任务，打开就能看效果
- **以后**：接入你自己的 Agent（LangChain、AutoGPT、CrewAI 或自己写的都行）

## 怎么用

### 线上

1. 打开 [oxelia51.com](https://oxelia51.com) 注册登录
2. 工具菜单里找到 AgentCanvas
3. 选个示例任务，看 Agent 怎么一步步跑

### 桌面版

1. 去 [GitHub Releases](https://github.com/XiaoleC05/AgentCanvas/releases) 下载 `AgentCanvas.exe`
2. 双击运行，不需要联网
